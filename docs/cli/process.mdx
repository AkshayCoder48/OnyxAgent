title: Process Management
description: Manage OnyxAgent process lifecycle with onyx commands
---

Process management commands control the OnyxAgent background process. These commands are only available in the terminal.

## start

Start the OnyxAgent service. Runs as a background daemon by default and automatically tails logs.

```bash
onyx start
```

**Options:**

| Option | Description |
| --- | --- |
| `-f`, `--foreground` | Run in foreground, not as a background daemon |
| `--no-logs` | Don't tail logs after starting |

## stop

Stop the running OnyxAgent service.

```bash
onyx stop
```

## restart

Restart the OnyxAgent service (stop then start).

```bash
onyx restart
```

**Options:**

| Option | Description |
| --- | --- |
| `--no-logs` | Don't tail logs after restart |

## update

Update code and restart the service. Automatically performs:

1. Pull latest code (`git pull`)
2. Stop current service
3. Update Python dependencies
4. Reinstall CLI
5. Start service

```bash
onyx update
```

<Warning>
  If `git pull` fails (e.g., uncommitted local changes), the update aborts and the service remains unaffected.
</Warning>

## status

Check OnyxAgent service status, including process info, version, and current model/channel configuration.

```bash
onyx status
```

## logs

View service logs.

```bash
onyx logs
```

**Options:**

| Option | Description | Default |
| --- | --- | --- |
| `-f`, `--follow` | Continuously tail log output | No |
| `-n`, `--lines` | Show last N lines | 50 |

Examples:

```bash
# View last 100 lines
onyx logs -n 100

# Continuously tail logs
onyx logs -f
```

## install-browser

Install Playwright and Chromium browser for the [browser tool](/tools/browser).

```bash
onyx install-browser
```

<Tip>
  Only needed when using browser tools (web browsing, screenshots, etc.).
</Tip>

## run.sh Compatibility

If Onyx CLI is not installed, you can use `run.sh` to manage the service:

| onyx command | run.sh equivalent |
| --- | --- |
| `onyx start` | `./run.sh start` |
| `onyx stop` | `./run.sh stop` |
| `onyx restart` | `./run.sh restart` |
| `onyx update` | `./run.sh update` |
| `onyx status` | `./run.sh status` |
| `onyx logs` | `./run.sh logs` |

<Note>
  The `onyx` command is recommended — it provides cleaner syntax and richer features. It is automatically installed via the one-click install script.
</Note>
